# 实现说明

## 点对点即时信息实现方式

```
ClientA               Server               ClientB
        ----发送1---->
        <--确认收到2--
                     存数据库
                             ----发送3---->
                             <--确认收到4--
                     存数据库
        <--告知收到5--
        --确认告知6-->
                             <--确认已读7--
                             --收到确认8-->
                     存数据库
        <--告知已读9--
        --确认告知10-->
```

数据库 msg_status：

ClientA->ClientB类别：
0 已发送至服务端 （第1步后即为此类型，待ClientB接收）
1 已发送至ClientB（第4步后即为此类型，待告知ClientA）
